Fonts and font availability

The text on the cover page has to be set in fonts supporting all the scripts used in COP, which is fairly large and includes Latin/Western European, Cyrillic, Greek, Chinese, Arabic and Hebrew.

Running the sheet_test.sh, will execute a java class that dumps the names of all available fonts.

1. Commercial fonts

There are fonts available for the purpose, like Arial Unicode MS and Lucida Sans Unicode, both of which are available as commercial software from Microsoft.

Here is a book containing text in Arabic script. Everything in Arial Unicode:

http://code-01.kb.dk/svn/cop-pdf/trunk/test/object61539.en.pdf

2. Multiple fonts, open source

There are also quite a few other solutions that require the use of multiple fonts depending upon language. This is the same one is using a font family DejaVue, which is open source

http://code-01.kb.dk/svn/cop-pdf/trunk/test-oss-fonts/dejavue_arabic.pdf

However, with Chinese, we have to add another font

http://code-01.kb.dk/svn/cop-pdf/trunk/test-oss-fonts/dejavue_grf_chi_chinese.pdf

Here we are using two fonts. Both of which are open source, so there is no cost for fonts but I had to correct the document since the language of the title was incorrectly set to Danish.

3. Singel OSS font

I've found a fairly complete font which is said to be GPL. It should in principle be able to do it all.

http://code-01.kb.dk/svn/cop-pdf/trunk/test-oss-fonts/Code2000_chinese.pdf

Problem is that it is written by a mr James Kass as shareware software. The price is set to USD 5 but the author disappeared from the Internet 2010 and since then it isn't known how to use Code2000 legally.

http://en.wikipedia.org/wiki/Code2000
